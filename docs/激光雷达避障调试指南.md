# 激光雷达避障调试指南

## 概述

本功能在 `feature/lidar-obstacle-avoidance` 分支上开发，使用 Docker 隔离 Livox Mid360 驱动环境，通过话题重定向实现避障仲裁，**不修改任何现有代码**。

### 话题流

```
现有控制节点（web_gui / ue_bridge / car_gui）
    ──> /cmd_vel_input  (CMD_VEL_TOPIC 覆盖)

Docker 容器（livox_ros_driver2 + pointcloud_to_laserscan）
    ──> /livox/lidar (PointCloud2)
    ──> /scan (LaserScan)

宿主机 obstacle_stopper.py
    订阅 /cmd_vel_input + /scan
    ──> /cmd_vel (底盘)
```

### 避障逻辑

| 前方最近距离 | 行为 |
|------------|------|
| > 3.0m | 透传原始指令，无干预 |
| 0.5m ~ 3.0m | 线速度按比例减速，保留转向 |
| < 0.5m | **刹停**，允许后退和原地转向脱困 |
| /scan 超时 | 退化为透传（fail-open） |

## 快速启动

### 第一次使用（需要构建 Docker 镜像）

```bash
cd ~/campusCar-old-chassis

# 1. 使用 lidar profile 启动全栈（将 cmd_vel 重定向到 cmd_vel_input）
CAR_PROFILE=old-orange-pi-orbbec-lidar ./scripts/launch_all.sh

# 2. 启动激光雷达栈（首次会构建镜像，需要几分钟）
./scripts/lidar_start.sh
```

### 日常使用

```bash
# 全栈已启动后，单独启动/停止激光雷达栈
./scripts/lidar_start.sh
./scripts/lidar_stop.sh

# 检查状态
./scripts/lidar_check.sh
```

## Mid360 网络配置

Mid360 通过以太网连接，**不是 USB 串口**。

### 默认 IP 配置

- Mid360 默认 IP：`192.168.1.177`
- NUC 需要在连接 Mid360 的网卡上配置 `192.168.1.2`

### 配置网卡 IP（宿主机）

```bash
# 查看网卡列表
ip link show

# 假设连接 Mid360 的网卡是 eth1（根据实际情况修改）
sudo ip addr add 192.168.1.2/24 dev eth1
sudo ip link set eth1 up

# 验证连通性
ping 192.168.1.177
```

### 让容器自动配置网卡 IP

```bash
# 在 lidar_start.sh 中指定网卡名
LIVOX_NET_IFACE=eth1 ./scripts/lidar_start.sh
```

### 修改 Mid360 IP（如需要）

编辑 `docker/lidar/config/mid360.json`，修改 `lidar_configs[0].ip` 字段。

## 参数调整

所有参数在 `config/profiles/old-orange-pi-orbbec-lidar.env` 中配置，修改后重启激光雷达栈生效。

### 避障参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `OBSTACLE_STOP_DIST` | 0.5m | 刹停距离 |
| `OBSTACLE_WARN_DIST` | 3.0m | 开始减速距离 |
| `OBSTACLE_FRONT_ANGLE_DEG` | 30° | 前方检测扇区半角 |
| `OBSTACLE_ENABLED` | 1 | 0=禁用（透传），1=启用 |
| `OBSTACLE_SCAN_TIMEOUT` | 2.0s | /scan 超时退化为透传 |

### 点云转换参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `SCAN_MIN_HEIGHT` | -0.3m | 垂直过滤下限 |
| `SCAN_MAX_HEIGHT` | 0.3m | 垂直过滤上限 |
| `SCAN_RANGE_MIN` | 0.1m | 最小检测距离 |
| `SCAN_RANGE_MAX` | 50.0m | 最大检测距离 |

### Mid360 安装位置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `LIDAR_X/Y/Z` | 0/0/0.2m | 相对于 base_link 的位置 |
| `LIDAR_ROLL/PITCH/YAW` | 0/0/0 | 相对于 base_link 的姿态 |

## 调试命令

### 查看激光雷达数据

```bash
# 查看点云话题
ros2 topic hz /livox/lidar

# 查看 LaserScan 话题
ros2 topic hz /scan
ros2 topic echo /scan --once

# 查看避障节点输出
ros2 topic echo /cmd_vel
```

### 查看日志

```bash
# 避障节点日志（关键词：[避障]）
tail -f data/logs/obstacle_stopper.log

# Docker 容器日志
docker logs -f campuscar_lidar

# 构建日志
cat data/logs/lidar_docker_build.log
```

### 测试避障功能

```bash
# 1. 手动发布前进指令到 /cmd_vel_input
ros2 topic pub /cmd_vel_input geometry_msgs/msg/Twist \
    "{linear: {x: 0.3}, angular: {z: 0.0}}" --rate 10

# 2. 在前方放置障碍物（距离 < 3.0m）
# 3. 观察 /cmd_vel 输出是否减速/停车
ros2 topic echo /cmd_vel

# 4. 查看避障日志
tail -f data/logs/obstacle_stopper.log
```

### 禁用避障（透传模式）

```bash
# 方式一：重启时禁用
OBSTACLE_ENABLED=0 ./scripts/lidar_start.sh

# 方式二：直接停止避障节点（底盘将不再收到 /cmd_vel）
./scripts/lidar_stop.sh
```

## 常见问题

### /scan 话题不出现

1. 检查 Docker 容器是否运行：`docker ps | grep campuscar_lidar`
2. 检查容器日志：`docker logs campuscar_lidar`
3. 检查 Mid360 网络：`ping 192.168.1.177`
4. 检查 /livox/lidar 是否出现：`ros2 topic list | grep livox`

### 避障节点启动失败

1. 检查 ROS2 环境：`ros2 topic list`
2. 检查日志：`cat data/logs/obstacle_stopper.log`
3. 手动运行调试：`python3 src/obstacle_stopper.py`

### 底盘不响应控制

1. 确认使用了 lidar profile：`echo $CMD_VEL_TOPIC`（应为 `/cmd_vel_input`）
2. 确认避障节点在运行：`pgrep -f obstacle_stopper.py`
3. 确认 /cmd_vel 有数据：`ros2 topic hz /cmd_vel`

### 恢复到无激光雷达模式

```bash
# 停止激光雷达栈
./scripts/lidar_stop.sh

# 使用原始 profile 重启全栈
./scripts/launch_all.sh
# 或
CAR_PROFILE=old-orange-pi-orbbec ./scripts/launch_all.sh
```

## 后期扩展：VFH 绕障

当前实现只做刹停保护。后期绕障扩展只需修改 `src/obstacle_stopper.py`：

1. 在 `_on_scan()` 中构建极坐标直方图
2. 在 `_tick()` 中添加 `avoid` 状态：找最优通行方向，发布转向指令
3. 不需要修改其他任何文件

参考算法：VFH（Vector Field Histogram），无需 TF 树、costmap、全局规划器。
