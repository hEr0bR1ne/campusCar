# campusCar 硬件复用指南

## 目标

这个分支把工程拆成“通用上层能力 + 硬件 profile”：

- 通用层：UE 桥接、RTK、路径录制/导航、GUI、键盘控制、rosbridge、RTSP/HLS/MJPEG 推流、一键启动/停止/检查。
- 硬件层：底盘启动方式、底盘 IP/SSH、底盘控制话题、相机驱动、相机 ROS topic、相机依赖和进程清理规则。

## 目录

```text
config/
  robot.env                       # 通用默认值和 profile loader
  robot.local.env                 # 本机覆盖，Git 忽略
  profiles/
    campus_car.env                # 当前旧车硬件 profile
    campus_car.local.env.example  # 密码本地覆盖模板
    template.env                  # 新车 profile 模板
```

## 常用命令

默认使用旧车：

```bash
./scripts/launch_all.sh
./scripts/check_all.sh
./scripts/stop_all.sh
```

指定新车：

```bash
./scripts/deploy_dependencies.sh --profile car_2
./scripts/launch_all.sh --profile car_2
./scripts/check_all.sh --profile car_2
./scripts/stop_all.sh --profile car_2
```

也可以用环境变量：

```bash
ROBOT_PROFILE=car_2 ./scripts/launch_all.sh
```

## 接入新底盘

复制模板：

```bash
cp config/profiles/template.env config/profiles/car_2.env
```

重点填写：

```bash
CHASSIS_START_MODE="ssh_ros2"     # ssh_ros2 / local_command / skip
CAR_IP="192.168.100.2"
CAR_USER="robot"
CAR_LAUNCH_CMD="source ~/ros2_ws/install/setup.bash && ros2 launch <pkg> <launch>.py"
CHASSIS_NODE_CHECK_PATTERN="<node_name_keyword>"
CMD_VEL_TOPIC="/cmd_vel"
```

密码放到 `config/profiles/car_2.local.env`：

```bash
CAR_PASS="..."
SUDO_PASS="..."
```

如果新底盘驱动跑在 NUC 本机：

```bash
CHASSIS_START_MODE="local_command"
LOCAL_CHASSIS_LAUNCH_CMD="source /opt/ros/humble/setup.bash && ros2 launch <pkg> <launch>.py"
```

如果底盘由外部系统负责启动：

```bash
CHASSIS_START_MODE="skip"
CHASSIS_REQUIRED="0"
```

## 接入新相机

重点填写：

```bash
CAMERA_START_MODE="ros2_launch"   # ros2_launch / command / skip
CAMERA_LAUNCH_PACKAGE="<camera_pkg>"
CAMERA_LAUNCH_FILE="<camera_launch>.py"
IMAGE_TOPIC="/camera/color/image_raw"
CAMERA_PROCESS_PATTERN="<camera_process_or_launch_name>"
CAMERA_DEPENDENCY_MODE="custom"
```

如果相机有独立工作区：

```bash
CAMERA_SETUP_FILES=(
    "/home/hkust-gz-nuc/<camera_ws>/install/setup.bash"
)
```

如果相机启动不能用 `ros2 launch`，改用命令模式：

```bash
CAMERA_START_MODE="command"
CAMERA_LAUNCH_CMD="source /opt/ros/humble/setup.bash && <camera_start_command>"
```

如果相机已经由外部服务启动：

```bash
CAMERA_START_MODE="skip"
IMAGE_TOPIC="/new_camera/image"
```

## 保持不变的上层接口

除非新车的 ROS topic 不兼容，否则 UE 负责人和上层应用继续使用：

```text
ws://<NUC_IP>:9090
http://<NUC_IP>:8888/robot_cam/index.m3u8
http://<NUC_IP>:8080/
/U2RTopic_Command
/R2UTopic_Pos
/R2UTopic_Text
```

换车优先只改 profile。只有当新底盘不是 `geometry_msgs/Twist` 或新相机没有 ROS2 `sensor_msgs/Image` 输出时，再新增真正的代码适配器。
