# 速度闭环调试指南

> 适用版本：`hardware/old-orange-pi-orbbec` 分支，2026-04-29 改造后的 `src/ue_bridge.py`

---

## 背景

本次改造为 `ue_bridge.py` 的运动控制加入了两项能力：

1. **方向控制**（Forward / TurnBackward）加入速度 PI 闭环 + 可选 `duration` 参数
2. **坐标导航**改为外环位置减速曲线 + 内环速度 PI 闭环

`/odom` 有数据时走闭环；`/odom` 超时或缺失时自动退化为开环，不影响基本运动。

---

## 第一步：重启 ue_bridge，加载新代码

全栈已由 systemd 服务在后台运行，但 `ue_bridge.py` 是旧版本。需要手动重启它：

```bash
# 找到旧进程 PID
pgrep -a python3 | grep ue_bridge

# 杀掉旧进程（替换 <PID>）
kill <PID>

# 前台启动新版本，方便实时看日志
cd ~/campusCar-old-chassis
source /opt/ros/humble/setup.bash
python3 src/ue_bridge.py
```

启动成功后应看到：
```
速度闭环：kp=0.8 ki=0.3 odom超时=1.0s
```

---

## 第二步：确认 /odom 有数据

另开终端：

```bash
source /opt/ros/humble/setup.bash
ros2 topic hz /odom
```

底盘在线时应有稳定频率（通常 10~50 Hz）。没有数据时速度闭环自动退化为开环，日志里不会报错。

---

## 第三步：测试方向控制速度环

### 3a. 命令行模拟 Forward 指令

```bash
source /opt/ros/humble/setup.bash
ros2 topic pub --once /U2RTopic_Command std_msgs/String \
  '{data: "{\"commandType\":\"move\",\"commandParams\":{\"destination\":\"Forward\",\"speed\":\"30\"}}"}'
```

在 `ue_bridge.py` 终端里，每 1 秒打印一行速度环日志：
```
[速度环] v_ref=0.30 v_actual=0.24 error=0.06 integral=0.12 output=0.31
```

### 3b. 测试 duration 参数（2 秒后自动停车）

```bash
ros2 topic pub --once /U2RTopic_Command std_msgs/String \
  '{data: "{\"commandType\":\"move\",\"commandParams\":{\"destination\":\"Forward\",\"speed\":\"30\",\"duration\":2.0}}"}'
```

2 秒后应看到：
```
UE 方向指令超时，已自动停车
```

> **注意**：实际超时 = `min(duration, UE_DIRECTION_TIMEOUT_SEC)`。默认 `UE_DIRECTION_TIMEOUT_SEC=0.8`，所以发 `duration=2.0` 时实际超时仍是 0.8s。如果要测试更长的 duration，先把 `config/robot.env` 里的 `UE_DIRECTION_TIMEOUT_SEC` 改大（例如 `5.0`），再重启 `ue_bridge.py`。

---

## 第四步：测试坐标导航减速曲线

从 UE 发坐标导航指令，或命令行模拟（替换为附近真实坐标）：

```bash
ros2 topic pub --once /U2RTopic_Command std_msgs/String \
  '{data: "{\"commandType\":\"move\",\"commandParams\":{\"destination\":{\"x\":113.12345,\"y\":22.12345},\"speed\":\"40\"}}"}'
```

接近目标时（距离 < `UE_NAV_DECEL_DIST=1.5m`），日志里 `v_ref` 应逐渐从目标速度降到 `UE_NAV_MIN_SPEED=0.08`：
```
[导航速度环] dist=1.20m v_ref=0.22 v_actual=0.19 error=0.03 integral=0.08 output=0.23
[导航速度环] dist=0.60m v_ref=0.14 v_actual=0.13 error=0.01 integral=0.09 output=0.14
```

## 调参说明

所有参数在 `config/robot.env` 里修改，改完重启 `ue_bridge.py` 即生效（不需要重启全栈）。

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `UE_SPEED_KP` | `0.8` | 速度环比例增益。响应太慢就调大，振荡就调小 |
| `UE_SPEED_KI` | `0.3` | 速度环积分增益。稳态误差大就调大，积分饱和就调小 |
| `UE_SPEED_ODOM_TIMEOUT` | `1.0` | `/odom` 超过此秒数未更新则退化为开环 |
| `UE_NAV_MIN_SPEED` | `0.08` | 导航最低速度（m/s），防止接近目标时速度过低失控 |
| `UE_NAV_DECEL_DIST` | `1.5` | 距目标多少米开始减速 |
| `UE_DIRECTION_TIMEOUT_SEC` | `0.8` | 方向指令最长持续时间（安全看门狗） |

### 典型调参场景

**速度响应慢，实际速度追不上目标**
→ 增大 `UE_SPEED_KP`（例如 `1.2`）

**速度有振荡/超调**
→ 减小 `UE_SPEED_KP` 或 `UE_SPEED_KI`

**接近目标时冲过头**
→ 减小 `UE_NAV_DECEL_DIST`（更早开始减速）或减小 `UE_NAV_MIN_SPEED`

**停车后还在缓慢漂移**
→ 检查 `UE_ARRIVE_THRESHOLD_M`（默认 `0.5m`），可适当调大

## 退化为开环的判断

日志里如果没有 `[速度环]` 或 `[导航速度环]` 行，说明在走开环。原因：
- `/odom` 话题没有数据（底盘未连接或 ROS 节点未启动）
- `/odom` 数据超过 `UE_SPEED_ODOM_TIMEOUT` 秒未更新

开环下运动仍然正常，只是没有速度反馈修正。
