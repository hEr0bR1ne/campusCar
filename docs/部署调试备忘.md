## 一、服务启动

```bash
cd ~/campusCar
./scripts/stop_all.sh    # 先清掉旧进程
./scripts/launch_all.sh  # 启动全套服务
```

`launch_all.sh` 会一起启动 RTK 串口驱动、rosbridge TCP/BSON 和 RTK/UE 数据桥。`/R2UTopic_Pos` 默认按 `UE_PUBLISH_RATE=1.0` 固定 1Hz 发给 UE，终端里的 `[R2U TX]` 是实际发送日志。当前终端也会实时显示带时间戳的 `[rtk-driver]`、`[rosbridge]`、`[rtk-ue]`、`[ue-last]` 输出；其中 `[ue-last]` 会每秒重复显示上一条 UE 指令，直到收到下一条。需要只写日志、不刷终端时：

```bash
LIVE_RTK_LOGS=0 ./scripts/launch_all.sh
```

检查 UE 坐标输出是否稳定 1Hz：

```bash
source ~/campusCar/config/robot.env && source "$ROS_SETUP"
ros2 topic hz /R2UTopic_Pos
```

单独启动 UE 指令桥接：

```bash
source ~/campusCar/config/robot.env
source "$ROS_SETUP"
python3 ~/campusCar/src/ue_bridge.py
```

单独打开小车控制台 GUI：

```bash
cd ~/campusCar
./scripts/open_car_gui.sh
```

## 五、不通过 UE 测试指令

### 先启动监控窗口

```bash
# 窗口1：监控发出的速度指令
ros2 topic echo /cmd_vel

# 窗口2：监控回复消息
ros2 topic echo /R2UTopic_Text
```

### 方向控制测试

```bash
source ~/campusCar/config/robot.env && source "$ROS_SETUP"

# 前进
ros2 topic pub --once /U2RTopic_Command std_msgs/msg/String \
  '{"data": "{\"commandId\":\"010\",\"commandType\":\"move\",\"RobotId\":\"Robot\",\"RobotType\":\" \",\"commandParams\":{\"destination\":\"Forward\",\"speed\":\"30\"}}"}'

# 后退
ros2 topic pub --once /U2RTopic_Command std_msgs/msg/String \
  '{"data": "{\"commandId\":\"010\",\"commandType\":\"move\",\"RobotId\":\"Robot\",\"RobotType\":\" \",\"commandParams\":{\"destination\":\"TurnBackward\",\"speed\":\"30\"}}"}'

# 左转
ros2 topic pub --once /U2RTopic_Command std_msgs/msg/String \
  '{"data": "{\"commandId\":\"010\",\"commandType\":\"move\",\"RobotId\":\"Robot\",\"RobotType\":\" \",\"commandParams\":{\"destination\":\"TurnLeft\",\"speed\":\"30\"}}"}'

# 右转
ros2 topic pub --once /U2RTopic_Command std_msgs/msg/String \
  '{"data": "{\"commandId\":\"010\",\"commandType\":\"move\",\"RobotId\":\"Robot\",\"RobotType\":\" \",\"commandParams\":{\"destination\":\"TurnRight\",\"speed\":\"30\"}}"}'

# 停车
ros2 topic pub --once /U2RTopic_Command std_msgs/msg/String \
  '{"data": "{\"commandId\":\"010\",\"commandType\":\"move\",\"RobotId\":\"Robot\",\"RobotType\":\" \",\"commandParams\":{\"destination\":\"Stop\",\"speed\":\"0\"}}"}'
```

方向指令由 `ue_bridge.py` 按 `UE_DIRECTION_RATE_HZ` 持续补发到 `/cmd_vel`。默认 `UE_DIRECTION_TIMEOUT_SEC=0.8` 秒没收到新方向指令就停车；UE 长按时建议 5~10Hz 重复发送，松开时发送 `speed=0` 或 `Stop`。

### 坐标导航测试

```bash
# 1. 先查当前 GPS 位置
ros2 topic echo /fix --once

# 2. 把目标设成附近几米的点（修改 x/y 为实际坐标）
```

## 六、ue_bridge.py 可调参数

文件位置：`~/campusCar/src/ue_bridge.py`

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `MAX_LINEAR_SPEED` | `1.0` m/s | speed=100 对应的最大速度 |
| `MAX_ANGULAR_SPEED` | `1.0` rad/s | 最大转向角速度 |
| `ARRIVE_THRESHOLD_M` | `0.5` m | 到达判定距离，GPS精度不足时调大 |
| `HEADING_KP` | `1.2` | 转向比例系数，实车测试后调整 |
| `HEADING_TOLERANCE` | `0.15` rad | 朝向误差容忍（约8.6°） |
| `UE_DIRECTION_RATE_HZ` | `10` Hz | 方向指令补发频率 |
| `UE_DIRECTION_TIMEOUT_SEC` | `0.8` s | UE 方向指令超时停车时间 |
| `TANK_TURN_MODE` | `angular` | `angular` 保持零线速度原地转向；`experimental_xz` 为实验性 X/Z 反向混控，实车确认前不要默认启用 |
| `TANK_TURN_SIDE_SPEED_SCALE` | `1.0` | 仅 `experimental_xz` 使用 |
| `TANK_TURN_MIN_SIDE_SPEED` | `0.10` m/s | 仅 `experimental_xz` 使用 |
| `TANK_TURN_MAX_SIDE_SPEED` | `1.0` m/s | 仅 `experimental_xz` 使用 |
| `UE_COORD_MODE` | `auto` | `gps` 只收经纬度，`local` 强制 UE 场景坐标转换，`auto` 自动判断 |
| `UE_LOCAL_ORIGIN_LAT/LON` | 空 | UE 场景锚点对应的真实 RTK 经纬度 |
| `UE_LOCAL_ORIGIN_X/Y` | `0` | UE 场景锚点坐标 |
| `UE_UNITS_PER_METER` | `100` | UE 单位到米，默认 100 cm = 1 m |
| `UE_LOCAL_ROTATION_DEG` | `0` | UE +X 轴相对真实正东的逆时针角度 |
| `UE_LOCAL_X_SIGN/Y_SIGN` | `1` | 某个轴方向相反时设为 `-1` |

---

## 七、坐标导航工作原理

1. 收到 `destination: {"x": ..., "y": ...}` 后，先按 `UE_COORD_MODE` 解析成目标经纬度
2. 计算当前位置到目标的 ENU 方位角
3. 原地转向，直到朝向误差 < `HEADING_TOLERANCE`
4. 直行，并在每个控制周期继续做小角度修正
5. 距目标 < `ARRIVE_THRESHOLD_M` 时停车，向 UE 回复 `arrived`
6. 新指令到来时自动取消当前导航任务

## 九、日志位置

所有运行日志写入 `~/campusCar/data/logs/`：

| 文件 | 对应服务 |
|------|---------|
| `camera.log` | 相机节点（Orbbec 或 Hikrobot/camera_aravis2） |
| `nmea_navsat_driver.log` | RTK 串口驱动 |
| `rosbridge.log` | rosbridge TCP/BSON |
| `ue5_bridge.log` | u2r_r2u_bridge |
| `mediamtx.log` | RTSP/HLS 服务器 |
| `ffmpeg_rtsp.log` | ffmpeg 推流 |
| `rtsp_server.log` | RTSP 推流节点 |
| `mjpeg_server.log` | MJPEG 预览服务 |
| `car_gui.log` | 控制 GUI 启动输出 |
| `ue_bridge.log` | UE 指令桥接 |

实时输出不会替代日志文件；终端显示的 RTK/rosbridge 内容仍会保存在上面的对应日志里。

相机启动慢时优先看 `camera.log`。Orbbec 相机重点看 `Start device cost`、`Initialize device cost` 和 `usb connect type`；如果显示 `USB2.1`，相机初始化和首帧通常会明显变慢。海康 `MV-CS016-10GC` 重点先跑 `./scripts/hikrobot_camera_probe.sh`，确认 `arv-tool-0.8` 和 `camera_aravis2 camera_finder` 能看到相机；再检查 NUC 网卡和相机是否在同一网段、PoE/外部供电是否正常、`HIKROBOT_CAMERA_GUID` 是否填了探测到的 GUID 或 IP。软件侧默认会复用已经出图的相机进程（`REUSE_CAMERA=1`），只有需要强制重启相机时再用 `REUSE_CAMERA=0`。`RESET_ORBBEC_USB=1` 仅建议在 Orbbec USB 设备疑似卡死时使用。

海康相机当前预置走 `CAMERA_MODE=hikrobot_gige_aravis`，图像最终仍发布到统一的 `IMAGE_TOPIC=/camera/color/image_raw`。如果 `camera.log` 里报 `PixelFormat` 不支持，先把 `HIKROBOT_PIXEL_FORMAT` 从 `BGR8` 改成 `RGB8` 试一次；如果仍失败，用：

```bash
arv-tool-0.8 --name=<GUID或IP> features
```

查看相机实际支持的 GenICam 参数名和值。
