# campusCar 新底盘快速启动指南

当前分支是 STM32 双 UART 新底盘专用分支，默认 profile 是 `stm32_hoverboard_4wd`。旧底盘 `campus_car` profile、旧 Docker 入口和旧相机路径不在这个分支支持范围内。

## 前置条件

硬件连接：

- STM32 前驱动器 TTL/USB 串口连接 NUC
- STM32 后驱动器 TTL/USB 串口连接 NUC
- Hikrobot `MV-CS016-10GC` GigE 相机连接到 NUC 网口或同网段交换机
- RTK 模块 USB 连接 NUC（可选，无 RTK 时跳过）
- 车轮架空或断开电机负载，急停可用

首次部署：

```bash
cd ~/campusCar-new-chassis
./scripts/deploy_dependencies.sh
```

如果要隔离 ROS2/驱动依赖，优先使用 Docker：

```bash
cd ~/campusCar-new-chassis
./scripts/docker_build.sh
./scripts/docker_run_stm32.sh
```

详细见 `docs/Docker部署指南.md`。

## 配置

通用配置在 `config/robot.env`，新底盘默认 profile 在：

```bash
config/profiles/stm32_hoverboard_4wd.env
```

每台 NUC 的本地覆盖不要写进 Git，放到被忽略的 local 文件：

```bash
nano ~/campusCar-new-chassis/config/profiles/stm32_hoverboard_4wd.local.env
```

建议实车上把前后两个串口改成稳定的 `/dev/serial/by-id/...`：

```bash
HOVERBOARD_FRONT_DEVICE="/dev/serial/by-id/<front-driver>"
HOVERBOARD_REAR_DEVICE="/dev/serial/by-id/<rear-driver>"
```

卖家协议允许 `steer/speed` 到 `[-1000,1000]` RPM，但工程默认 `HOVERBOARD_COMMAND_LIMIT_RPM=50`。确认车轮架空、方向正确、急停可用之前，不要提高这个值。

Hikrobot 相机到货后先探测：

```bash
./scripts/hikrobot_camera_probe.sh
```

如果看到相机 GUID 或 IP，可写入本地覆盖：

```bash
HIKROBOT_CAMERA_GUID="<GUID或IP>"
```

## 启动顺序

1. 探测新底盘依赖和串口：

```bash
./scripts/stm32_hoverboard_probe.sh
```

2. 探测 Hikrobot 相机：

```bash
./scripts/hikrobot_camera_probe.sh
```

3. 确认硬件安全后启动全栈：

```bash
./scripts/launch_all.sh
```

`launch_all.sh` 会启动 STM32 底盘驱动、相机、RTSP/HLS/MJPEG 视频、RTK 数据流、rosbridge TCP/BSON、UE 数据桥和控制 GUI。缺少前后任一底盘串口时，脚本会停止，这是为了避免新底盘接线不明确时误发控制。

## UE 数据流

| 数据流 | 话题/端口 |
|------|------|
| RTK 原始定位 | `/fix` |
| UE 位置 JSON | `/R2UTopic_Pos` |
| UE 文本回复 | `/R2UTopic_Text` |
| UE 指令入口 | `/U2RTopic_Command` |
| rosbridge TCP/BSON | `9090` |

`/R2UTopic_Pos` 默认按 `UE_PUBLISH_RATE=1.0` 固定 1Hz 发送给 UE。启动终端显示的 `[R2U TX]` 就是该话题的发送日志。

启动成功后终端会打印所有服务地址：

```text
GPS/UE5:  <NUC_IP>:9090
视频流:   rtsp://<NUC_IP>:8554/robot_cam
UE(HLS):  http://<NUC_IP>:8888/robot_cam/index.m3u8
浏览器:   http://<NUC_IP>:8080/
```

## 停止和检查

```bash
./scripts/stop_all.sh
./scripts/check_all.sh
```

Docker 内运行：

```bash
./scripts/docker_run_stm32.sh -- ./scripts/check_all.sh
./scripts/docker_run_stm32.sh -- ./scripts/stm32_hoverboard_probe.sh
```

## 键盘控制

一键启动后的“小车控制台”窗口已经内置键盘控制。点击窗口后，长按 `w/a/s/d` 或方向键移动，松开运动键即停车；界面按钮也需要长按，松开即停车；`space`/`x` 急停，`r/f` 调整线速度，`t/g` 调整角速度，`q/z` 同时缩放线速度和角速度。

单独打开控制台：

```bash
./scripts/open_car_gui.sh
```

终端键盘控制：

```bash
./scripts/keyboard_control.sh
```

## 单独启动模块

所有命令需先 source 环境：

```bash
source ~/campusCar-new-chassis/config/robot.env
source "$ROS_SETUP"
for setup_file in "${CHASSIS_SETUP_FILES[@]}" "${CAMERA_SETUP_FILES[@]}"; do
    [ -f "$setup_file" ] && source "$setup_file"
done
```

| 模块 | 命令 |
|------|------|
| STM32 底盘探测 | `./scripts/stm32_hoverboard_probe.sh` |
| STM32 底盘驱动 | `./scripts/stm32_hoverboard_start.sh` |
| Hikrobot 相机探测 | `./scripts/hikrobot_camera_probe.sh` |
| 控制 GUI | `python3 ~/campusCar-new-chassis/src/car_gui.py` |
| MJPEG 预览 | `python3 ~/campusCar-new-chassis/src/mjpeg_server.py` |
| RTSP 推流 | `python3 ~/campusCar-new-chassis/src/rtsp_server.py` |
| UE 指令桥接 | `python3 ~/campusCar-new-chassis/src/ue_bridge.py` |
| GPS 路径录制 | `python3 ~/campusCar-new-chassis/src/rtk_tools/path_recorder.py` |
| GPS 航点导航 | `python3 ~/campusCar-new-chassis/src/rtk_tools/gps_navigator.py --waypoints <路径文件.json>` |
