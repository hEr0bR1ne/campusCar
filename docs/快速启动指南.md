# campusCar 快速启动指南

## 前置条件

**硬件连接**
- NUC 通过交换机连接小车底盘（`192.168.100.2`）
- Orbbec 相机 USB 连接 NUC
- RTK 模块 USB 连接 NUC（可选，无 RTK 时跳过）

**软件依赖**（首次部署时安装）
```bash
cd ~/campusCar-old-chassis
./scripts/deploy_dependencies.sh
```

脚本会安装 rosbridge、RTK/视频/Python 运行依赖、Orbbec 相机依赖，安装 Orbbec udev 规则，并构建 `~/campusCar-old-chassis/orbbec_ws`。

**mediamtx**（RTSP/HLS 服务器，需在 PATH 中）
```bash
# deploy_dependencies.sh 会在缺失时自动下载到 /usr/local/bin/mediamtx
mediamtx --version
```

## 配置（换车时修改）

只需编辑一个文件：

```bash
nano ~/campusCar-old-chassis/config/robot.env
```

关键配置项：

```bash
CAR_IP="192.168.100.2"      # 小车底盘 IP
CAR_USER="bingda"            # 小车 SSH 用户名
CAR_PASS="bingda"            # 小车 SSH 密码
SUDO_PASS="1"                # NUC 本机 sudo 密码
CAR_BASE_TYPE="4WD"          # 底盘硬件自报类型
CAR_LAUNCH_CMD="export BASE_TYPE=${CAR_BASE_TYPE}; source ~/ros2_ws/install/setup.bash && ros2 launch base_control_ros2 base_control.launch.py"
```

## 一键启动

```bash
cd ~/campusCar-old-chassis
./scripts/launch_all.sh
```

一键启动会同时拉起 RTK 数据流：

| 数据流 | 话题 |
|------|------|
| RTK 原始定位 | `/fix` |
| UE 位置 JSON | `/R2UTopic_Pos` |
| UE 文本回复 | `/R2UTopic_Text` |
| UE 指令入口 | `/U2RTopic_Command` |
| rosbridge TCP/BSON | `9090` |

`/R2UTopic_Pos` 默认按 `UE_PUBLISH_RATE=1.0` 固定 1Hz 发送给 UE；启动终端显示的 `[R2U TX]` 就是该话题的发送日志。默认启用静止锁点：`/odom` 速度小于等于 `RTK_POSITION_HOLD_SPEED_THRESHOLD_MPS=0.03` 时，发给 UE 的经纬度保持缓存值，避免静止时 RTK 漂移；车辆移动后才用新的 RTK 坐标覆盖缓存。控制台 `/R2UTopic_Pos 发给 UE` 区域的 `坐标源` 会显示 `移动更新`、`静止锁定` 或里程计异常时的实时 RTK 回退状态。默认情况下，启动终端会实时显示 RTK/rosbridge/UE 关键输出，并带时间戳前缀，例如 `[2026-04-27 14:24:00][rtk-ue] ...`。`[ue-last]` 会每秒显示上一条 UE 指令，在收到下一条前保持重复显示上一条内容。需要安静启动时可使用：

```bash
LIVE_RTK_LOGS=0 ./scripts/launch_all.sh
```

验证 UE 坐标发送频率：

```bash
source ~/campusCar-old-chassis/config/robot.env
source "$ROS_SETUP"
ros2 topic hz /R2UTopic_Pos
```

UE 指令桥接也会随 `launch_all.sh` 启动，直接监听 `/U2RTopic_Command`：

- `destination` 为 `"Forward"`、`"TurnLeft"`、`"TurnRight"`、`"TurnBackward"`、`"Stop"` 时，桥接节点按 `UE_DIRECTION_RATE_HZ=10` 持续补发 `/cmd_vel`，默认 `UE_DIRECTION_TIMEOUT_SEC=0.8` 秒没有新指令就自动停车。
- `destination` 为 `{"x": ..., "y": ...}` 时，默认按 x=经度、y=纬度导航；如果 UE 发的是场景坐标，在 `config/robot.env` 中把 `UE_COORD_MODE=local` 并配置 `UE_LOCAL_ORIGIN_LAT/LON`、`UE_UNITS_PER_METER`、`UE_LOCAL_ROTATION_DEG` 等转换参数即可，UE 发送的 JSON 格式不用改。

启动成功后终端会打印所有服务地址：

```
  网页控制: http://<NUC_IP>:8088/
  GPS/UE5:  <NUC_IP>:9090
  视频流:   rtsp://<NUC_IP>:8554/robot_cam
  UE(HLS):  http://<NUC_IP>:8888/robot_cam/index.m3u8
  浏览器:   http://<NUC_IP>:8080/
```

## 网页控制台

全栈启动后，手机、平板或另一台电脑只要和 NUC 在同一网络，就可以直接打开：

```text
http://<NUC_IP>:8088/
```

网页控制台会显示相机画面、底盘/RTK/UE 状态、最近 UE 指令、关键日志，并可以直接长按方向按钮或键盘 `w/a/s/d` 控制 `/cmd_vel`。网页端有 `WEB_CONTROL_TIMEOUT_SEC=0.45` 秒死区保护：浏览器断连、松手或页面失焦后会自动停车。视频仍复用 `MJPEG_PORT=8080` 的现有流，所以 UE 的 HLS/RTSP 接入格式不受影响。

单独启动网页控制台：

```bash
cd ~/campusCar-old-chassis
./scripts/open_web_gui.sh
```

网页控制台端口可在 `config/robot.env` 里改：

```bash
WEB_GUI_PORT=8088
```

## 开机自启

安装并启用全栈开机自启服务：

```bash
cd ~/campusCar-old-chassis
./scripts/install_autostart_service.sh
```

服务名是 `campuscar-old-chassis.service`，默认用 `LIVE_RTK_LOGS=0 START_WEB_GUI=1 START_CONTROL_GUI=0 ./scripts/launch_all.sh` 启动，也就是开机后自动启动全栈和网页控制台，但不弹出桌面 GUI。立即启动服务可用：

```bash
./scripts/install_autostart_service.sh --start-now
```

查看状态：

```bash
systemctl --user status campuscar-old-chassis.service --no-pager
```

关闭自启：

```bash
./scripts/install_autostart_service.sh --disable
```

## 桌面 GUI / 键盘控制

`launch_all.sh` 现在默认启动网页控制台，不再自动弹出桌面 Tk GUI。需要在 NUC 本机桌面上打开旧版窗口时：

```bash
START_CONTROL_GUI=1 ./scripts/launch_all.sh
```

桌面 GUI 和网页控制台都带键盘控制，并会显示 `/fix` 原始 RTK 坐标、`/R2UTopic_Pos` 发给 UE 的实时坐标、rosbridge 连接状态、UE 相关话题 pub/sub 状态、以及 `/U2RTopic_Command` 最近一条 UE 指令。点击控制界面后，长按 `w/a/s/d` 或方向键移动，单独按住 `a/d` 或左右方向键会发送零线速度原地转向指令，组合键 `w+a`、`w+d`、`s+a`、`s+d` 保持行进中转向逻辑，松开运动键即停车；`space`/`x` 急停。

底盘状态面板里的 `设为正北` 会把当前 `/imu` 或 `/odom` 朝向记录为正北，并写回 `config/robot.env` 的 `VEHICLE_HEADING_OFFSET_DEG`。现场校准时先把车头摆到真实正北，再点击一次；网页/桌面 GUI 会立即生效，RTK/UE 数据桥也会自动读取新的配置值。

单独打开小车控制台：

```bash
cd ~/campusCar-old-chassis
./scripts/open_car_gui.sh
```

桌面上也会有 `campusCar - 小车控制台` 快捷方式。

需要单独用终端键盘控制时，另开一个终端：

```bash
cd ~/campusCar-old-chassis
./scripts/keyboard_control.sh
```

终端控制模式下，按住 `w/a/s/d` 或方向键运动，单独 `a/d` 发送零线速度原地转向指令，松开后短暂停止；`space`/`x` 停车，`q` 退出。

```bash
source ~/campusCar-old-chassis/config/robot.env

[ "$CAMERA_MODE" = "orbbec" ] && [ -f "$ORBBEC_SETUP" ] && source "$ORBBEC_SETUP"
```

## 相机模式

默认配置仍使用当前车的 Orbbec：

```bash
CAMERA_MODE=orbbec ./scripts/launch_all.sh
```

新车的海康工业相机 `MV-CS016-10GC` 走 GigE Vision/GenICam，优先用 ROS2 Humble 的 `camera_aravis2`：

```bash
# 首次部署会由 deploy_dependencies.sh 安装；也可单独安装
sudo apt install ros-humble-camera-aravis2 aravis-tools aravis-tools-cli

# 相机到货后先单独探测，确认网口/IP/GUID
./scripts/hikrobot_camera_probe.sh

# 如果探测到 GUID 或 IP，建议固定设备再启动全栈
CAMERA_MODE=hikrobot_gige_aravis HIKROBOT_CAMERA_GUID=<GUID或IP> ./scripts/launch_all.sh
```

海康 profile 会把 `camera_aravis2` 的图像话题 remap 到项目统一的 `IMAGE_TOPIC=/camera/color/image_raw`，所以 GUI、MJPEG、RTSP/HLS 和 UE 侧视频链路不需要改。常用可调项在 `config/robot.env` 里：

```bash
HIKROBOT_PIXEL_FORMAT=BGR8
HIKROBOT_WIDTH=1440
HIKROBOT_HEIGHT=1080
HIKROBOT_FPS=30.0
HIKROBOT_GEV_PACKET_SIZE=9000  # 只有网卡和相机都启用 jumbo frame 时再设
```

如果 `BGR8` 不被相机当前固件接受，先试 `RGB8`；如果仍失败，再用 `arv-tool-0.8 --name=<GUID或IP> features` 或海康 MVS 查看实际支持的 `PixelFormat` 名称。

## 启动流程说明

`launch_all.sh` 按以下顺序启动：

1. 清理旧进程，释放端口
2. ping 检查小车网络连通性
3. SSH 远程启动小车底盘节点（若未运行）
4. 验证 ROS2 DDS 话题发现（`/cmd_vel`）
5. 按 `CAMERA_MODE` 启动或复用相机节点，视频服务先订阅并等待首帧
6. 启动 mediamtx RTSP/HLS 服务器 + ffmpeg 推流
7. 启动 MJPEG HTTP 预览服务，并提前打开网页控制台
8. 自动检测 RTK 串口，启动 nmea_navsat_driver + rosbridge TCP/BSON + u2r_r2u_bridge
9. 启动 UE 指令桥接（`ue_bridge.py`）
10. 如前面未打开网页控制台，则启动 `car_web_gui.py`；桌面 Tk GUI 仅在 `START_CONTROL_GUI=1` 时启动

RTK 串口未插时，`/fix` 数据源会自动跳过，rosbridge 和 UE 数据通道仍会启动。RTK/rosbridge 的实时输出仍会同步写入 `data/logs/`。

常用加速/排查开关：

```bash
# 默认值：复用已经出图的相机，避免反复等待相机初始化
REUSE_CAMERA=1 ./scripts/launch_all.sh

# 需要强制重启相机时使用
REUSE_CAMERA=0 ./scripts/launch_all.sh

# 只有怀疑 USB 设备卡死时才重置 Orbbec USB
RESET_ORBBEC_USB=1 REUSE_CAMERA=0 ./scripts/launch_all.sh

# GUI 默认优先读取本机 MJPEG 流，失败时仍保留 ROS 图像订阅
CAR_GUI_CAMERA_SOURCE=auto ./scripts/launch_all.sh
```
