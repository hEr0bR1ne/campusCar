# campusCar 快速启动指南

## 前置条件

**硬件连接**
- NUC 通过交换机连接小车底盘（`192.168.100.2`）
- Orbbec 相机 USB 连接 NUC
- RTK 模块 USB 连接 NUC（可选，无 RTK 时跳过）

**软件依赖**（首次部署时安装）
```bash
cd ~/campusCar
./scripts/deploy_dependencies.sh
```

脚本会安装 rosbridge、RTK/视频/Python 运行依赖，并按当前 hardware profile 安装相机依赖。默认 `campus_car` profile 会安装 Orbbec 相机依赖、Orbbec udev 规则，并构建 `~/campusCar/orbbec_ws`。

**mediamtx**（RTSP/HLS 服务器，需在 PATH 中）
```bash
# deploy_dependencies.sh 会在缺失时自动下载到 /usr/local/bin/mediamtx
mediamtx --version
```

## 配置（换车时修改）

通用配置在 `config/robot.env`，每台车的底盘和相机差异放在 `config/profiles/<profile>.env`。默认旧车 profile 是：

```bash
nano ~/campusCar/config/profiles/campus_car.env
```

密码和每台 NUC 的本地覆盖不要写进 Git，放到被忽略的 local 文件：

```bash
cp ~/campusCar/config/profiles/campus_car.local.env.example \
   ~/campusCar/config/profiles/campus_car.local.env
nano ~/campusCar/config/profiles/campus_car.local.env
```

新车接入时复制模板：

```bash
cp ~/campusCar/config/profiles/template.env ~/campusCar/config/profiles/car_2.env
nano ~/campusCar/config/profiles/car_2.env
```

## 一键启动

```bash
cd ~/campusCar
./scripts/launch_all.sh
```

指定新车 profile：

```bash
./scripts/launch_all.sh --profile car_2
```

一键启动会同时拉起 RTK 数据流：

| 数据流 | 话题 |
|------|------|
| RTK 原始定位 | `/fix` |
| UE 位置 JSON | `/R2UTopic_Pos` |
| UE 文本回复 | `/R2UTopic_Text` |
| UE 指令入口 | `/U2RTopic_Command` |
| rosbridge TCP/BSON | `9090` |

`/R2UTopic_Pos` 默认按 `UE_PUBLISH_RATE=1.0` 固定 1Hz 发送给 UE；启动终端显示的 `[R2U TX]` 就是该话题的发送日志。默认情况下，启动终端会实时显示 RTK/rosbridge/UE 关键输出，并带时间戳前缀，例如 `[2026-04-27 14:24:00][rtk-ue] ...`。`[ue-last]` 会每秒显示上一条 UE 指令，在收到下一条前保持重复显示上一条内容。需要安静启动时可使用：

```bash
LIVE_RTK_LOGS=0 ./scripts/launch_all.sh
```

验证 UE 坐标发送频率：

```bash
export ROBOT_PROFILE=campus_car
source ~/campusCar/config/robot.env
source "$ROS_SETUP"
ros2 topic hz /R2UTopic_Pos
```

UE 指令桥接也会随 `launch_all.sh` 启动，直接监听 `/U2RTopic_Command`：

- `destination` 为 `"Forward"`、`"TurnLeft"`、`"TurnRight"`、`"TurnBackward"`、`"Stop"` 时，桥接节点按 `UE_DIRECTION_RATE_HZ=10` 持续补发 `/cmd_vel`，默认 `UE_DIRECTION_TIMEOUT_SEC=0.8` 秒没有新指令就自动停车。
- `destination` 为 `{"x": ..., "y": ...}` 时，默认按 x=经度、y=纬度导航；如果 UE 发的是场景坐标，在 `config/robot.env` 中把 `UE_COORD_MODE=local` 并配置 `UE_LOCAL_ORIGIN_LAT/LON`、`UE_UNITS_PER_METER`、`UE_LOCAL_ROTATION_DEG` 等转换参数即可，UE 发送的 JSON 格式不用改。

启动成功后终端会打印所有服务地址：

```
  GPS/UE5:  <NUC_IP>:9090
  视频流:   rtsp://<NUC_IP>:8554/robot_cam
  UE(HLS):  http://<NUC_IP>:8888/robot_cam/index.m3u8
  浏览器:   http://<NUC_IP>:8080/
```

## 键盘控制

一键启动后的“小车控制台”窗口已经内置键盘控制，并会显示 `/fix` 原始 RTK 坐标、`/R2UTopic_Pos` 发给 UE 的实时坐标、rosbridge 连接状态、UE 相关话题 pub/sub 状态、以及 `/U2RTopic_Command` 最近一条 UE 指令。点击窗口后，长按 `w/a/s/d` 或方向键移动，组合键 `w+a`、`w+d`、`s+a`、`s+d` 斜向移动，松开运动键即停车；界面按钮也需要长按，松开即停车；`space`/`x` 急停，`r/f` 调整线速度，`t/g` 调整角速度，`q/z` 同时缩放线速度和角速度。

单独打开小车控制台：

```bash
cd ~/campusCar
./scripts/open_car_gui.sh
```

桌面上也会有 `campusCar - 小车控制台` 快捷方式。

需要单独用终端键盘控制时，另开一个终端：

```bash
cd ~/campusCar
./scripts/keyboard_control.sh
```

终端控制模式下，按住 `w/a/s/d` 或方向键运动，松开后短暂停止；`space`/`x` 停车，`q` 退出。

## 桌面快捷方式（可选）

将 `.desktop` 文件安装到桌面，双击即可操作：

```bash
# 安装到桌面
cp ~/campusCar/*.desktop ~/Desktop/
chmod +x ~/Desktop/*.desktop

# 或安装到应用菜单
cp ~/campusCar/*.desktop ~/.local/share/applications/
```

四个快捷方式：

| 图标名 | 功能 |
|--------|------|
| campusCar - 启动全栈 | 等同于 `launch_all.sh` |
| campusCar - 停止全栈 | 等同于 `stop_all.sh` |
| campusCar - 状态检查 | 等同于 `check_all.sh` |
| campusCar - 路径录制 | 启动 GPS 路径录制工具 |

## 单独启动各模块

所有命令需先 source 环境：

```bash
source ~/campusCar/config/robot.env
source "$ROS_SETUP"
for setup_file in "${CAMERA_SETUP_FILES[@]}"; do
    [ -f "$setup_file" ] && source "$setup_file"
done
```

| 模块 | 命令 |
|------|------|
| 控制 GUI | `python3 ~/campusCar/src/car_gui.py` |
| MJPEG 预览 | `python3 ~/campusCar/src/mjpeg_server.py` |
| RTSP 推流 | `python3 ~/campusCar/src/rtsp_server.py` |
| UE 指令桥接 | `python3 ~/campusCar/src/ue_bridge.py` |
| GPS 路径录制 | `python3 ~/campusCar/src/rtk_tools/path_recorder.py` |
| GPS 航点导航 | `python3 ~/campusCar/src/rtk_tools/gps_navigator.py --waypoints <路径文件.json>` |

## 启动流程说明

`launch_all.sh` 按以下顺序启动：

1. 清理旧进程，释放端口
2. 按 profile 检查底盘网络连通性
3. 按 profile 启动底盘节点（SSH、local command 或跳过）
4. 验证 ROS2 DDS 话题发现（默认 `/cmd_vel`）
5. 按 profile 启动或复用相机节点，视频服务先订阅并等待首帧
6. 启动 mediamtx RTSP/HLS 服务器 + ffmpeg 推流
7. 启动 MJPEG HTTP 预览服务，并提前打开控制 GUI
8. 自动检测 RTK 串口，启动 nmea_navsat_driver + rosbridge TCP/BSON + u2r_r2u_bridge
9. 启动 UE 指令桥接（`ue_bridge.py`）
10. 如前面未打开 GUI，则启动控制 GUI（`car_gui.py`）

RTK 串口未插时，`/fix` 数据源会自动跳过，rosbridge 和 UE 数据通道仍会启动。RTK/rosbridge 的实时输出仍会同步写入 `data/logs/`。

常用加速/排查开关：

```bash
# 默认值：复用已经出图的相机，避免反复等待相机初始化
REUSE_CAMERA=1 ./scripts/launch_all.sh

# 需要强制重启相机时使用
REUSE_CAMERA=0 ./scripts/launch_all.sh

# 只有怀疑 USB 设备卡死时才重置相机 USB；具体 USB ID 来自当前 profile
RESET_CAMERA_USB=1 REUSE_CAMERA=0 ./scripts/launch_all.sh

# GUI 默认优先读取本机 MJPEG 流，失败时仍保留 ROS 图像订阅
CAR_GUI_CAMERA_SOURCE=auto ./scripts/launch_all.sh
```
